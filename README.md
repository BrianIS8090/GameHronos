# Хроно-Метро: Петля Энтропии

Интерактивная нарративная игра в браузере. Вагон метро летит в перегоне между станциями «Проспект Хроноса» и «Октябрьская-Аномальная», в салоне — бомба, террорист и три минуты до взрыва. Каждая ваша смерть откатывает время в начало вагона, но память о пройденном остаётся: то, что игрок узнал в одной петле, доступно во всех последующих. Чтобы разорвать петлю, нужно собрать семь улик и сделать единственный правильный выбор за один цикл.

---

## Как запустить

Игра использует ES-модули (`import` из `js/game.js`, `js/rules.js`, `js/text-data.js`, `js/audio.js`). Все современные браузеры блокируют загрузку модулей по протоколу `file://` — если просто дважды кликнуть по `index.html`, начальный экран откроется, но кнопка «СИНХРОНИЗИРОВАТЬ» не среагирует на нажатие, потому что обработчик не успевает привязаться.

**Поэтому локальный HTTP-сервер обязателен.** Любой из вариантов подойдёт:

```
# Python 3 (есть из коробки на Linux / macOS, ставится в Windows)
python -m http.server 8000

# Node.js (npx без установки)
npx serve .

# PHP
php -S localhost:8000

# VS Code — расширение «Live Server», правый клик по index.html → Open with Live Server
```

После запуска откройте в браузере `http://localhost:8000/` (для serve по умолчанию `http://localhost:3000/`).

Прогресс сохраняется в `localStorage` ключами `chrono_memories_v2`, `chrono_opened_paths_v2`, `chrono_loop_v2`. Кнопка «Сбросить память» в подвале игры стирает всё и начинает с первой петли.

### Мобильная версия

Вёрстка адаптирована под планшеты и телефоны:

- На экранах **≤ 1024 px** боковая панель архива ужимается, отступы становятся компактнее.
- На экранах **≤ 768 px** интерфейс перестраивается в одну колонку: нарратив сверху, архив снизу с собственным скроллом. Шапка переходит на две строки, тач-цели увеличиваются.
- На экранах **≤ 480 px** скрываются подписи статов (остаются только значения), подвал становится вертикальным, мини-игра keypad ужимается до 56 px кнопок.
- Альбомная ориентация телефонов с высотой **≤ 500 px** дополнительно сжимает схему вагона.

Чтобы поиграть с телефона, поднимите сервер на компьютере (`python -m http.server 8000`) и откройте `http://АДРЕС_ПК_В_ЛОКАЛЬНОЙ_СЕТИ:8000/` — телефон и ноутбук должны быть в одной Wi-Fi сети.

## Геймплей

- **27 уникальных путей** через три сцены — `i-j-k`, где каждый индекс от 0 до 2.
- **7 улик**, каждая открывается своим путём (и иногда влияет на доступные действия в следующем цикле).
- **3 минуты на петлю.** Истёкший таймер = автоматический рестарт.
- **Три типа исходов:**
  - Фатальная концовка — кнопка RESTART, новый цикл с сохранённой памятью.
  - Стелс-концовка — кнопка CONTINUE, продолжение осмотра вагона в той же сессии, таймер не сбрасывается.
  - Истинная концовка — петля разорвана.
- **Хроно-резонанс:** при наличии улики в сцене 1 появляется специальная подсказка о том, что игрок уже знает в этой петле.
- **Интерактивная схема вагона** — клик по подсвеченной зоне открывает Хроно-анализатор с информацией об объекте, основанной на собранных уликах.
- **Финальный кодовый замок:** в истинной концовке `2-2-2` запускается мини-игра — подобрать 3-значный код из 9 кнопок (6 из них «сломаны» и подсвечиваются). До 9 попыток.

Минимальное прохождение до истинной концовки — **8 циклов**.

## Тестирование

Все правила разблокировки живут в чистой функции `applyUnlocks(path, memoriesBefore, stateBefore)` в `js/rules.js`. Эта же функция импортируется и в `game.js`, и в тестах — поэтому runtime и тесты физически не могут разойтись.

Запуск из корня проекта:

```
node js/test-runner.mjs
```

Один прогон проверяет:

- 54 базовых smoke-теста (27 путей × {память пустая, память полная}) — каждая концовка имеет непустой текст.
- 19 расширенных аудитов: согласованность текста и выдаваемых улик, достижимость каждой улики из чистого старта, симуляция оптимального прохождения, провальные ветки финала, балансировка HTML, регрессии по найденным ранее багам.

Ожидаемый результат — `Critical: 0, High: 0, Medium: 0, Low: 0`.

В UI игры также есть встроенная панель «Авто-тестирование» (кнопка в подвале) — она перебирает все 27 путей с обоими режимами памяти и показывает результат прямо в окне.

## Архитектура решений

- **Единая модель правил.** Раньше unlock-логика была размазана между `game.js` и тестовой копией в `test-audit.mjs`; теперь — одна экспортируемая функция. Snapshot памяти и стейта берётся до любых мутаций, поэтому порядок условий внутри не влияет на выдачу улик.
- **Каскадные улики.** Любое вскрытие рюкзака (ветка `0-2-x`) автоматически открывает `bomb_found` — игрок не может узнать жёлтый провод, не узнав сначала про саму бомбу.
- **Контроль террориста как предусловие.** Улика `yellow_wire` выдаётся только если террорист подтверждённо под контролем: либо `terrorist_know && soldier_trust` (знаем цель и доверие Алексея), либо `terroristNeutralized` (стелс-захват в текущей сессии).
- **Keypad защищён от обхода.** При отмене/провале мини-игры `pathHistory` откатывается с `[2, 2, 2]` до `[2, 2]`, иначе игра считала бы любой следующий выбор как `2-2-2` без решения замка. Таймер при возврате возобновляется методом `_resumeTimer()`.
- **Глитч-эффект безопасен для HTML.** Случайное искажение в `typewriterEffect` пропускает токены, содержащие `<`, `>` или `&`, чтобы не разламывать `<strong>` и `<em>` в текстах концовок.

## Что игроку нужно знать (без спойлеров)

1. Каждая смерть — это информация. Если концовка фатальная, прочитайте подсказку в её конце — там сказано, что попробовать в другой ветке.
2. Стелс-концовки не сбрасывают таймер. Используйте оставшееся время, чтобы продолжить осмотр.
3. Память сохраняется между циклами. Если в сцене 1 появился «Хроно-резонанс» — значит, эта улика уже у вас.
4. Не все ветки безопасны после того, как улика собрана. Например, после знакомства с террористом ветка «Осмотр» превращается в схватку.
5. Если в финальной мини-игре с кодом застряли — нажмите «Отмена», вернётесь к выбору действий в кабине без потери прогресса.
