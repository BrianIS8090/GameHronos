# Хроно-Метро — Полная карта состояний

> Эта карта строится **из исходного кода**, а не из задумок. Все условия — фактические, проверенные `grep`-ом в `js/game.js` и `js/text-data.js`.

## 1. Глоссарий состояний

| Тип | Что хранит | Где живёт | Сбрасывается |
|-----|-----------|-----------|--------------|
| `memories` | 7 boolean (улики) | `localStorage.chrono_memories_v2` | Только при `initiateChronoCollapse()` |
| `state.openedPaths` | Set строк `i-j-k` | `localStorage.chrono_opened_paths_v2` | То же |
| `state.loop` | Номер цикла | `localStorage.chrono_loop_v2` | То же |
| `state.cabinCode` | 3 случайные цифры | Только в RAM | Каждый `startLoop()` |
| `state.terroristNeutralized` | bool | Только в RAM | Каждый `startLoop()` |
| `state.pathHistory` | `[i, j, k]` | Только в RAM | Каждый `startLoop()` |

`terroristNeutralized` устанавливается в `showEnding()` для путей `1-1-1` и `1-1-2` при наличии `terrorist_know`. Используется в `getAvailableChoices` (запрет выбора i=1) и в `getEndingText` (особая концовка для `0-2-2`).

## 6. applyUnlocks — единственный источник правды

После рефакторинга (см. `docs/audit-findings.md`) логика разблокировки живёт в `js/rules.js` как чистая функция. Snapshot памяти и стейта берётся ДО любых мутаций, поэтому порядок условий внутри не влияет на выдачу:

```js
// js/rules.js
export function applyUnlocks(path, memoriesBefore, stateBefore = {}) {
  const [i, j, k] = path;
  const hadTerroristKnow = !!memoriesBefore.terrorist_know;
  const hadGeneratorKey = !!memoriesBefore.generator_key;
  const hadSoldierTrust = !!memoriesBefore.soldier_trust;
  const terroristNeutralized = !!stateBefore.terroristNeutralized;

  // bomb_found — каскадно открывается при любом 0-2-x (вскрытие рюкзака).
  if (i === 0 && j === 1 && (k === 1 || k === 2)) unlock('bomb_found');
  if (i === 0 && j === 2) unlock('bomb_found');

  if ((i === 1 && j === 0 && k === 2) || (i === 1 && j === 2 && k === 0)) unlock('terrorist_know');
  if (i === 2 && j === 1 && (k === 0 || k === 1)) unlock('generator_key');

  // brake_broken через стоп-кран — только без terrorist_know (иначе ветка про прыжок).
  // Через люк (2-1-2) — только если уже добыт ключ.
  if ((i === 1 && j === 2 && k === 0 && !hadTerroristKnow) ||
      (i === 2 && j === 1 && k === 2 && hadGeneratorKey)) {
    unlock('brake_broken');
  }

  if ((i === 2 && j === 0 && k === 1) || (i === 2 && j === 2 && k === 1)) unlock('cabin_code');

  if ((i === 1 && j === 2 && k === 1 && !hadTerroristKnow) ||
      (i === 1 && j === 2 && k === 2 && hadTerroristKnow && !hadSoldierTrust)) {
    unlock('soldier_trust');
  }

  // yellow_wire — нужен подтверждённый контроль над террористом.
  const terroristControlled = (hadTerroristKnow && hadSoldierTrust) || terroristNeutralized;
  if (i === 0 && j === 2 && k === 2 && terroristControlled) unlock('yellow_wire');
}
```

`game.js:checkUnlocks` теперь — тонкая обёртка:

```js
checkUnlocks(i, j, k) {
  const { fired } = applyUnlocks([i, j, k], this.memories, this.state);
  fired.forEach((key) => this.unlockMemory(key));
}
```

Эта же функция импортируется в `js/test-audit.mjs` — так что тесты и игра используют один и тот же код.

## 7. Проверка истинной концовки 2-2-2 (порядок)

`getEndingText(2,2,2,...)` проверяет улики **в строгом порядке**:

1. `!(bomb_found && yellow_wire)` → "ТАЙНА МЕДНЫХ ЖИЛ"
2. `!terrorist_know` → "НЕИЗВЕСТНЫЙ ФАКТОР"
3. `!soldier_trust`:
   - `terroristNeutralized` → "ВРЕМЕННОЙ ТАЙМЕР"
   - иначе → "ДИСТАНЦИОННЫЙ СДВИГ"
4. `!generator_key` → "БЕЗИНСТРУМЕНТАЛЬНЫЙ СБОЙ"
5. `!brake_broken` → "СКОРОСТЬ КАТАСТРОФЫ"
6. `!cabin_code` → "СЛУЧАЙНЫЙ ВЗЛОМ"
7. Все 7 есть → **ИСТИННЫЙ РАССВЕТ**

> Игра требует **все 7 улик одновременно**. При победе обязательно проиграть мини-игру keypad — без `cabin_code` система блокирует попытку угадывания (см. пункт 6).

## 8. Минимальный достижимый путь к финалу

С учётом блокировки i=1 после `terrorist_know`:

| Цикл | Путь | Цель | Условие |
|------|------|------|---------|
| 1 | 0-1-1 | `bomb_found` | Чистый старт |
| 2 | 1-0-2 | `terrorist_know` | Без улик ветки i=1 |
| 3 | 2-1-0 / 2-1-1 | `generator_key` | — |
| 4 | 2-1-2 | `brake_broken` | Требует `generator_key` |
| 5 | 2-0-1 | `cabin_code` | — |
| 6 | 1-2-2 | `soldier_trust` | Требует `terrorist_know` |
| 7 | 0-2-2 | `yellow_wire` | Требует `soldier_trust` |
| 8 | 2-2-2 | **Победа** | Требует все 7 + угадать код |

**Итого: 8 циклов минимум.** (narrative-map.md ошибочно указывает 9.)

> Альтернатива: при чистом старте можно сделать `1-2-0` вторым циклом — получится одновременно `terrorist_know` + `brake_broken`. Но тогда фатальный путь работает только без `terrorist_know`, а если уже есть → текст vB про прыжок на террориста, и `brake_broken` выдаётся как несоответствующий сюжету бонус (см. дыра №1).

## 10. Keypad: откат и таймер

`startKeypadGame()` останавливает таймер. До рефакторинга 21.05.2026 это создавало две дыры:

1. **Отмена не откатывала `pathHistory`** — внутри оставалось `[2, 2, 2]`. Если игрок после отмены выбирал на сцене 3 `k=0`/`k=1`, новый выбор добавлялся четвёртым элементом, но `showEnding()` всё равно брал `k = pathHistory[2]` (то есть 2). Это позволяло достичь истинной концовки без решения keypad.
2. **Таймер не возобновлялся** после `_keypadCancel()` и `_keypadFail()` → BACK. Игрок мог зафиксировать время.

Сейчас:
- `_keypadCancel()` и `_keypadFail()` → BACK откатывают `pathHistory` с `[2, 2, 2]` до `[2, 2]`, восстанавливают `currentScene = 3` и вызывают `_resumeTimer()`.
- `_resumeTimer()` — общий метод, продолжает таймер от текущего `timeRemaining`. Используется и в `startLoop()`, и в CONTINUE-кнопке, и в keypad-возврате.
